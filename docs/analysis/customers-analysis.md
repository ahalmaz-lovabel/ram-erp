# تحليل قسم العملاء والصفقات

|            |                                                                                      |
| ---------- | ------------------------------------------------------------------------------------ |
| **النطاق** | إدارة العملاء، الصفقات، عروض الأسعار، الفواتير، الدفعات، التسليم، الضمانات، والتواصل |
| **الغرض**  | مواصفات قابلة للمراجعة قبل التصميم والتنفيذ                                          |
| **الحالة** | تحليل معتمد — مرجع بناء موديول customers                                             |

> **الفكرة**: جعل ملف العميل مرجعًا كاملًا لكل تعاملاته التجارية والمالية والتشغيلية، من التسجيل حتى التسليم والضمان وخدمة ما بعد البيع.

> **ملاحظة نطاق**: الكيانات الأساسية لموديول customers هي **العميل + جهات التواصل + الصفقات + سجل التواصل**. عروض الأسعار (§7) والفواتير/الدفعات (§8) وكشف الحساب (§9) والاستحقاقات (§10) والتسليم/الشحن (§12) والضمانات (§13) تُبنى في موديولاتها (quotes/invoices/accounting/shipping) وتظهر داخل ملف العميل عبر ربط.

## 1. الهدف

المركز الرئيسي لإدارة علاقة الشركة بالعميل: تسجيل العميل (فرد/مؤسسة)، جهات التواصل، الفرص البيعية، العروض، الفواتير، الدفعات، التسليم، الضمان، كشف الحساب.

## 2. صفحة إدارة العملاء

- عرض كل العملاء · بحث بـ (الاسم، الهاتف، WhatsApp، البريد، كود العميل).
- فلترة حسب **النوع**: فرد، مؤسسة، شركة، نادي، جيم، موزع.
- فلترة حسب **الحالة**: محتمل، نشط، غير نشط، عليه مديونية، مهم.
- عرض الرصيد الحالي وآخر تعامل/متابعة لكل عميل.
- جدول: كود، اسم، نوع، جهة التواصل الأساسية، هاتف/WhatsApp، المدينة، الحالة، إجمالي الفواتير/المدفوع/المتبقي، آخر تعامل، إجراءات (عرض/تعديل/أرشفة/حذف).

> **ضابط الحذف**: الحذف العادي = أرشفة (حفظ تاريخ العميل). الحذف النهائي بصلاحية إدارية فقط، **ويُمنع** إذا كان العميل مرتبطًا بفواتير أو دفعات أو مستندات رسمية.

## 3. ملف العميل (تبويبات)

نظرة عامة · البيانات الأساسية · جهات التواصل · الصفقات · عروض الأسعار · الفواتير · الدفعات · كشف الحساب · خطة الدفع والاستحقاقات · الأجهزة والمنتجات · التسليم والشحن · الضمانات · التواصل والمتابعة · المرفقات والمستندات.

## 4. البيانات الأساسية للعميل

- **كود العميل** واسم العميل.
- **نوع العميل**: شخص، مؤسسة، شركة، نادي، جيم، موزع.
- **حالة العميل**: محتمل، نشط، غير نشط، عليه مديونية، مهم.
- الهاتف الأساسي، WhatsApp، البريد، العنوان.
- المدينة/المنطقة، الدولة (إن وجدت).
- الرقم الضريبي والسجل التجاري (إن وجدا).
- **مصدر العميل**: إعلان، ترشيح، زيارة، اتصال، معرض، عميل قديم.
- ملاحظات عامة، الموظف المسؤول، تاريخ الإنشاء، آخر تحديث.

> **حفظ تاريخ المستندات**: تعديل بيانات العميل لاحقًا لا يعدّل المستندات القديمة الصادرة (الفواتير/الضمانات/محاضر التسليم تحفظ بيانات العميل وقت الإصدار — snapshot).

## 5. جهات التواصل (العميل قد يكون مؤسسة بأكثر من جهة)

- الاسم والمسمى الوظيفي · الهاتف وWhatsApp والبريد.
- تحديد **جهة التواصل الأساسية**.
- **القسم**: إدارة، مشتريات، حسابات، موقع، استلام.
- ملاحظات · فتح WhatsApp/بريد أو تسجيل متابعة مباشرة.

## 6. الصفقات (فرصة بيع/مشروع محتمل)

- **رقم الصفقة** واسمها والعميل المرتبط.
- جهة التواصل الرئيسية والموظف المسؤول.
- مصدر الصفقة و**نوعها**: بيع مباشر، تجهيز جيم، توريد فقط، توريد وتركيب، إضافة لاحقة.
- القيمة التقديرية وتاريخ الإغلاق المتوقع (إن وُجد).
- الحالة والملاحظات والملفات المرتبطة.

**حالات الصفقة**: عميل مهتم · جاري التواصل · تم إرسال عرض سعر · مرحلة التفاوض · مقبولة · مرفوضة · مؤجلة · تحوّلت لأمر بيع · ملغاة.

## 7. عروض الأسعار _(موديول quotes — يظهر في ملف العميل)_

يُنشأ من ملف العميل أو الصفقة، ويُحفظ كنسخة (snapshot) كما صدرت. حالات: مسودة، مرسل، تحت التعديل، مقبول، مرفوض، منتهي، محوّل لأمر بيع/فاتورة.

## 8. الفواتير والدفعات _(موديولات invoices/accounting)_

حالات الفاتورة: غير مدفوعة، مدفوعة جزئيًا، مدفوعة، متأخرة، ملغاة. نظام الدفع الافتراضي: 50% مقدم + 50% عند الاستلام (قابل للتعديل). الدفعة: تاريخ، قيمة، طريقة (كاش/تحويل/بطاقة/شيك)، الفاتورة المرتبطة، الموظف، إيصال.

## 9. كشف حساب العميل _(accounting)_

حركات مالية مرتبة (فواتير، دفعات، خصومات، مرتجعات، تسويات) — تاريخ، نوع، بيان، مدين، دائن، الرصيد بعد الحركة، المرجع.

## 10. خطة الدفع والاستحقاقات _(accounting)_

الفاتورة/الصفقة، تاريخ الاستحقاق، المبلغ، الحالة (قادم/مستحق/مدفوع/متأخر)، أيام التأخير، تنبيهات.

## 11. الأجهزة والمنتجات الخاصة بالعميل

عرض ما اشتراه العميل بالأكواد والكميات + الفاتورة المرتبطة وتاريخ التسليم وحالة الضمان/التسليم. المستندات القديمة تحفظ الكود وقت الإصدار.

## 12. التسليم والشحن _(موديول shipping)_

حالات: لم يتم التجهيز · جاهز · تم التسليم لشركة الشحن · في الطريق · تم التسليم للعميل · تم التركيب · مكتمل · مشكلة/نقص/تلف. محاضر تسليم (شركة شحن / تركيب داخلي) بثلاث نسخ وتوقيعات.

## 13. الضمانات _(shipping/after-sales)_

يبدأ بعد تأكيد التسليم. المدة الافتراضية **12 شهرًا** من التسليم الفعلي. شهادة ضمان برقم + بيانات العميل + الفاتورة + المنتجات المشمولة + الشروط.

## 14. التواصل والمتابعة (سجل داخل ملف العميل)

- تاريخ المتابعة و**نوع التواصل**: اتصال، WhatsApp، بريد، زيارة، اجتماع.
- الموظف المسؤول وجهة التواصل داخل العميل.
- ملخص ما حدث، الخطوة التالية، تاريخ المتابعة القادمة.

## 15. المرفقات والمستندات

عروض/فواتير/إيصالات/محاضر/شهادات/عقود/صور. كل مرفق: اسم، نوع، تاريخ الرفع، من رفعه، المستند/الصفقة المرتبطة، ملاحظات.

## 16. الصياغة النهائية

> قسم العملاء والصفقات هو المركز الرئيسي لإدارة علاقة الشركة بالعميل: تسجيل العميل (فرد/مؤسسة) مع أكثر من جهة تواصل، إدارة الصفقات والفرص، إصدار العروض وتحويلها لفواتير/أوامر بيع، متابعة الدفعات وكشف الحساب والاستحقاقات والتسليم ومحاضر الشحن والضمانات — لتكون صفحة العميل مرجعًا كاملًا لكل التعاملات.
