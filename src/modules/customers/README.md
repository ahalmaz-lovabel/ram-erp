# موديول: customers

## الوصف

العملاء وجهات التواصل والصفقات. مبني على `docs/analysis/customers-analysis.md`.

## الحالة

🟡 **النواة مكتملة** (العميل + جهات التواصل + الصفقات، مُتحقَّق e2e على Staging). سجل التواصل والمرفقات لاحقًا.

## النطاق المنفَّذ

| الكيان                                             | الحالة                        |
| -------------------------------------------------- | ----------------------------- |
| العميل (§4): بيانات أساسية + نوع/حالة/مصدر + مسؤول | ✅ create/update/archive      |
| جهات التواصل (§5): مع ضمان جهة أساسية واحدة        | ✅ add/update/remove          |
| الصفقات (§6): رقم تسلسلي + نوع/حالة/قيمة تقديرية   | ✅ create/update/changeStatus |

## يعتمد على

- `shared/permissions` · `shared/audit` · `shared/errors` · `shared/auth` (الهوية من الجلسة)
- **`shared/services/sequenceGenerator`** — رقم الصفقة تسلسلي آمن من التعارض (بُني مع هذا الموديول).

## يُستخدم من (لاحقًا)

- `quotes` / `invoices` — تُنشأ من العميل/الصفقة وتحفظ snapshot لبياناته وقت الإصدار.

## قرارات وملاحظات

- **الحذف = أرشفة** (§2، CLAUDE #16). الحذف النهائي بصلاحية إدارية ويُمنع مع وجود مستندات — يُطبَّق عند بناء الفواتير.
- **"عليه مديونية"** فلتر مشتق من الرصيد (accounting لاحقًا)، و**"مهم"** علم مستقل `isImportant`.
- **جهة تواصل واحدة أساسية** لكل عميل (يُضبط تلقائيًا عند التعيين).
- **جهة تواصل الصفقة** لازم تتبع نفس العميل (`CONTACT_WRONG_CUSTOMER`).
- **رقم الصفقة** عبر `sequenceGenerator` داخل نفس المعاملة (يتراجع لو فشل الإنشاء).
