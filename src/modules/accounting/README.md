# موديول: accounting

## الوصف

الحسابات — كشف حساب العميل والمستحقات. **طبقة قراءة/تجميع** فوق الفواتير
والدفعات (لا جداول خاصة بها في هذه المرحلة).

## الحالة

🟡 **شريحة القراءة مكتملة** (كشف الحساب + المستحقات + الأعمار، backend + واجهات).
عمليات الكتابة المالية (مرتجعات/تسويات/ردّ مبالغ) لاحقًا.

## مصدر التحليل ⚠️

لا يوجد مستند تحليل مستقل. مبني على المصادر المعتمدة الجزئية:

- **تحليل العملاء §9** (كشف الحساب): حركات مالية مرتّبة (فواتير، دفعات، خصومات،
  مرتجعات، تسويات) — تاريخ، نوع، بيان، مدين، دائن، الرصيد بعد الحركة، المرجع.
- **تحليل العملاء §10** (الاستحقاقات): الفاتورة/الصفقة، تاريخ الاستحقاق، المبلغ،
  الحالة (قادم/مستحق/مدفوع/متأخر)، أيام التأخير، تنبيهات.

## المُنفَّذ (شريحة القراءة)

| الميزة                   | الوصف                                                                                  |
| ------------------------ | -------------------------------------------------------------------------------------- |
| كشف حساب العميل (§9)     | تجميع الفواتير (مدين) والدفعات (دائن) برصيد جارٍ — محرك نقي مُختبَر `statementBuilder` |
| المستحقات والأعمار (§10) | المتبقّي على كل عميل مع تمييز المتأخّر (overdue) عبر `isOverdue` من invoices           |

## افتراضات مؤقتة

1. **الحركات الحالية**: فواتير + دفعات فقط. الخصومات والمرتجعات والتسويات
   (المذكورة في §9) تحتاج كيانات كتابة مستقلة — مؤجّلة لشريحة تالية.
2. **الأعمار**: تمييز ثنائي (متأخّر / غير متأخّر) بناءً على `dueDate`. تقسيم دقيق
   لفترات (0–30 / 31–60 / 60+ يوم) مؤجّل.
3. **الرصيد موجب** = مستحق على العميل. الفواتير الملغاة مستبعدة تمامًا.
4. **عند تساوي التاريخ** تُقدَّم الفاتورة على الدفعة في الترتيب.
5. **لا رصيد افتتاحي** (opening balance) — الكشف من أول فاتورة.
