# موديول: invoices

## الوصف

الفواتير والدفعات — تُنشأ مستقلة أو بتحويل عرض سعر مقبول، بنودها snapshot،
ولها دفعات وحالات سداد تُشتق آليًا.

## الحالة

🟡 **الباك-إند مكتمل** (النماذج + الخدمات + التحقق + الاختبارات + migration على
Staging). الواجهات (UI) شريحة تالية.

## مصدر التحليل ⚠️

**لا يوجد مستند تحليل مستقل للفواتير في `docs/analysis/`**. مبني على المصادر
المعتمدة الجزئية:

- **تحليل العملاء §8**: حالات الفاتورة (غير مدفوعة · مدفوعة جزئيًا · مدفوعة ·
  متأخرة · ملغاة)، نظام الدفع الافتراضي 50% مقدم + 50% عند الاستلام (قابل
  للتعديل)، والدفعة (تاريخ · قيمة · طريقة كاش/تحويل/بطاقة/شيك · إيصال · موظف).
- **تحليل المنتجات §17**: تحويل العرض لفاتورة ينقل بيانات المنتج والأسعار
  والتكلفة وقت البيع؛ الفواتير المدفوعة تغذّي إحصائيات المباع.
- **تحليل المستخدمين**: صلاحيات حساسة "إلغاء فاتورة" و"تسجيل أو حذف دفعة".

**كشف الحساب (§9) وخطة الاستحقاقات (§10)** يخصّان موديول `accounting` — ليسا هنا.

## افتراضات مؤقتة

1. **حالة السداد تُشتق آليًا** من `paidAmount` مقابل `grandTotal`
   (`derivePaymentStatus`، دالة نقية مُختبَرة): 0 ⇒ غير مدفوعة، جزئي ⇒ مدفوعة
   جزئيًا، مكتمل ⇒ مدفوعة.
2. **overdue لا تُخزَّن** — تُحسب عند العرض (`isOverdue`) من `dueDate`. التحويل
   التلقائي لحالة overdue مؤجّل (يحتاج مهمة مجدولة).
3. **التعديل** (بنود/خصم/ضريبة) مسموح فقط **قبل أول دفعة وقبل الإلغاء** (لا توجد
   حالة "مسودة" في التحليل — تُعامَل الفاتورة كمُصدَرة عند الإنشاء).
4. **التحويل من عرض** يتطلب أن يكون العرض **مقبولًا** ولم يُحوَّل (علاقة 1:1)،
   ويختم العرض بحالة `converted`.
5. **منع الدفع الزائد**: الدفعة لا تتجاوز المتبقّي.
6. **الإلغاء** ممنوع على فاتورة بها دفعات (يلزم حذف الدفعات أولًا) — منطق ردّ
   المبالغ (refund) يخصّ `accounting` لاحقًا.
7. **حذف الدفعة** مسموح كصلاحية حساسة (التحليل نصّ عليها) ويُعاد احتساب الحالة —
   استثناء موثّق من قاعدة "الأرشفة بدل الحذف" (CLAUDE #16).
8. **تاريخ الاستحقاق** حقل اختياري؛ نظام 50/50 مذكور كافتراضي لكن التطبيق يترك
   المبالغ مرنة (دفعات جزئية بأي قيمة ≤ المتبقّي).
9. **العملة واحدة** (جنيه مصري) حتى يُقرَّر تعدد العملات.
10. **الوصف** في البند snapshot يُترك null (المنتج لا يملك حقل وصف بعد).

## دورة الحياة

```
إنشاء (مستقلة أو من عرض مقبول) ▶ unpaid
  ── دفعة ─▶ partiallyPaid ── دفعة ─▶ paid
  (dueDate تجاوز + غير مدفوعة/جزئية) ▶ overdue (محسوب عند العرض)
  cancel (بلا دفعات) ▶ cancelled
```

## يعتمد على

- `shared/permissions` · `shared/audit` · `shared/errors` · `shared/auth`
- `shared/services/sequenceGenerator` — رقم الفاتورة (INV-00001) داخل المعاملة
- `shared/services/documentTotals` — محرك الإجماليات المشترك مع quotes
- `customers` · `products` · `quotes` (التحويل)

## يُستخدم من (لاحقًا)

- `accounting` — كشف الحساب، الاستحقاقات، ردّ المبالغ.
- ملف العميل — تبويبات الفواتير والدفعات.
